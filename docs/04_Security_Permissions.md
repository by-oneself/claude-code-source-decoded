# 在“放手去做”与“防范删库”之间：安全与权限鉴定系统

当一个能随意执行终端命令的 AI 拿到你的电脑权限，最令人担忧的就是“越权与破坏”。
如果每一条命令都要询问用户，系统就成了点读机毫无“主动代理（Agentic）”可言；如果毫无阻拦，系统分分钟会跑出一个 `rm -rf` 搞崩环境。

Anthropic 在 Claude Code `src/utils/permissions/` 等目录下构筑的代码，可以说是最令人拍案叫绝的一套双规鉴权（Heuristic + ML）防护罩。

---

## 1. 拦截策略的双通道

当你运行 `claude --yes` (免打扰自动模式) 时，如果你以为它真的不管什么指令都会跑，那就大错特错了。

### 第一层：强校验字符串正则黑名单 (Heuristic Layer)
在系统底层的沙盒处理层，硬编码了一套 `disallowed_commands`：
包含但不限于：修改本机高危配置、部分 git 强制洗牌操作。
如果 Agent 通过 BashTool 企图派发这类的指令，会直接通过传统拦截抛回一个本地生成的拒绝提示，模型连请求执行的机会都没有。

### 第二层：动态微型分类器 (Yolo Classifier)
这是全剧组最大亮点之一，代码主要在 `src/utils/permissions/yoloClassifier.ts`。

什么是 Yolo (You Only Look Once / 你只活一次) 模式？代表用户授予了高度信任。
但信任必须建立在评估机制上。当大模型抛出一条**不在黑白名单直接覆盖范围**内的模糊命令（例如 `curl -X POST 删除某个接口` 或某类深度系统调用），它会：
1. **触发影子判断**：本地调度逻辑会在毫秒内唤起另一个针对安全性做过微调（Fine-tuned）的、跑得极为快速的轻量 LLM 审查链路分类器模型（YOLO Model）。
2. **审查放行**：影子模型专门用于在后台极速审阅该操作指令的风险。如果其判决得分处于 Safe，则静默执行；若处于高危 Warning 区，则将原大语言模型直接挂起挂断（拦截流）。
3. **退回请求**：被拦截后，系统并非抛出系统错误，而是模拟成用户发给大模型一条消息：“这个操作可能具有危险性，我拒绝运行。请通过其他查询指令或向我详细报告原因再请求”。

## 2. 工具调用的独立权限沙盒

并不是所有的 Tool 都具有最高运行权限。
在 `src/constants/tools.ts` 中维护了细粒度的权限矩阵，例如：

```typescript
export const ALL_AGENT_DISALLOWED_TOOLS = new Set([
  TASK_OUTPUT_TOOL_NAME,
  EXIT_PLAN_MODE_V2_TOOL_NAME,
  ...
])
```

这确保了**哪怕发生黑客攻击般的 Prompt 逃逸/注入**，在一个被召唤出的子代理（Sub-Agent）环境中，它由于物理上被剥夺了退出计划模式和操作顶层任务调度器的权限钩子，这使得这个被注入的坏代理也只能在本地局部文件上搞破坏，无法“越狱”到顶层枢纽层篡改用户的宏观 Agent 进度控制层。
